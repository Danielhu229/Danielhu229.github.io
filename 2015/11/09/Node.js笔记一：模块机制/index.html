Node.js笔记一：模块机制

2015-11-09

1.CommonJS的模块规范

CommonJS

CommonJS规范为javascript提供了模块系统，标准接口，包管理器等超集，使得javascript可以具备像Python，Ruby和Java一样具备大型应用开发的能力

曾经，javascript是薄弱而不受待见的，在web1.0时代甚至被称为玩具语言，直到CommonJS和Node的出现以及后来各种前端框架和HTML5的发展，这一状况才逐渐得到改善，CommonJS作为一个规范为Node所采用，对javascript开发大型应用的指导作用是巨大的

利用CommonJS规范，开发者不仅可以写出javascript的富客户端应用，甚至能够编写服务器端的js程序(Node 就是一个很好的例子)以及其他大型程序

####主要规范

CommonJS对模块的定义十分简单,主要分为模块引用，模块定义和模块标识3个部分

#####1.模块引用
模块引用的示例代码如下

var math = require("math");

CommonJS规范存在 require() 方法，这个方法接受模块标示，这样可以引入一个模块的API到当前上下文当中

#####2.模块定义

CommonJS 提供了exports对象用于导出当前模块的方法或者变量
exports 是 module 对象的属性，而module代表模块自身
被导出的方法和变量必须挂载在exports对象上

exports.add = function(){
  var sum = 0, i = 0 args = arguments, l = args.length;
  while(i < 1){
    sum += args[i++];
  }
  return sum;
};

在另一个文件中通过require()方法引入模块后，就能调用自定义的属性和方法了

#####3.模块标识

模块标识就是传递给require()方法的参数，他必须是符合小驼峰命名的字符串，或者以 . 和 ..开头的相对路径，或者绝对路径。可以省略后缀名.js

###2.Node的模块实现

在Node中，模块分为两类，一是 核心模块 一是 文件模块

  • 核心模块是预编译好的二进制文件，Node启动时就已经被加载到内存中，所以不需要文件定位和编译执行
  • 文件模块是在运行时动态加载（解释），需要完整的路径分析，文件定位，编译执行过程，速度比核心模快慢

我们知道浏览器会通过缓存静态脚本文件以提高性能，Node同样会对 编译和执行之后的对象 进行缓存
实际上，Node最先加载的是核心模块的缓存，然后是文件模块的缓存，然后编译加载核心模块未加载部分，最后去编译加载文件模块未加载部分
